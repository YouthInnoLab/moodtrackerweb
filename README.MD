MoodTracker 🧠
💜AI-Powered Anonymous Student Wellbeing PlatformMoodTracker is a privacy-first, multimodal web application designed for the Build4Student Hackathon. It enables students to express their emotional state through text, voice, or imagery without the friction of account creation. By leveraging AI for sentiment analysis, the platform provides schools with aggregated, location-based insights to identify mental health trends across campus.🏗️ Design ArchitectureThe application follows a Decoupled Client-Side Architecture designed for high performance and offline-first capabilities (PWA).Frontend Framework: React 18+ (Vite-powered) for a responsive, component-based UI.Styling: Tailwind CSS using a utility-first approach for consistent design and rapid prototyping.AI Engine: Integration with Anthropic Claude API (via backend proxy or direct SDK) to process natural language, voice transcripts, and visual cues into emotional data points.Storage Layer: * Local: Browser Storage API (IndexedDB/LocalStorage) for temporary session persistence.Mocking: A custom storage-mock.js is implemented to simulate database interactions during development.Multimodal Inputs: * Web Audio API: For real-time voice recording.MediaDevices API: For camera access and photo capture.🔄 WorkflowThe user journey is designed to be completed in under 30 seconds to encourage high engagement:Input Selection: The student chooses a modality (Text, Voice, Photo, or Emoji).Contextual Tagging: The student selects their current location (e.g., Library, Gym, Cafeteria) to provide spatial context.AI Analysis: * Text/Voice is parsed for sentiment and intensity.Photos are analyzed for emotional markers.Anonymization: All PII (Personally Identifiable Information) is stripped. Only the mood, location, and timestamp are retained.Aggregation: Data is synced to the school’s dashboard as a part of a heat-map/trend analysis, ensuring no individual student can be identified.🧪 Test Procedure1.
